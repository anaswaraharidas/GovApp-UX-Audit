# GovApp-UX-Audit
A UX Audit and Case Study of the mParivahan app, focusing on reducing interaction cost, improving visual hierarchy, and streamlining digital document retrieval for better accessibility.
# Project Overview
This project presents a rapid User Experience (UX) audit of mParivahan, India’s primary digital platform for vehicle documentation. The objective of this study was to identify critical usability bottlenecks and propose a streamlined, modern interface to improve the efficiency of document retrieval for citizens.
# Key Pain Points Identified
High Interaction Cost: The process of accessing a saved Driving License (DL) requires excessive navigation through deep menus. This complexity increases user stress during time-sensitive situations, such as roadside document verification.
Visual Overload: The current home screen utilizes inconsistent iconography and redundant text, which contributes to significant cognitive load and makes navigation less intuitive.
Authentication Friction: Frequent session timeouts and re-verification requirements hinder immediate access, resulting in a suboptimal experience for an application designed for essential, on-the-go use.
# Proposed Design Solutions
Persistent Dashboard: I propose a "Digital Wallet" concept featuring a Quick Access card on the home screen. This would display primary DL and RC QR codes immediately upon launching the app to minimize retrieval time.
Simplified Information Architecture: Consolidating the existing home screen into four distinct categories—My Documents, Services, Traffic Violations, and Information—would clarify the user journey and reduce clutter.
Enhanced Visual Feedback: Implementing clear status indicators (such as color-coded validity markers) would allow users to verify document status at a glance without needing to parse fine-print details.
# Methodology
The audit was conducted using two primary frameworks:
Heuristic Evaluation: Assessing the interface against Nielsen’s 10 Usability Heuristics to identify standard design failures.
User Flow Mapping: Analyzing and measuring the specific sequence of actions required to complete the "View Document" task.
# Expected Impact
By reducing the "time-to-task" for critical document retrieval, these proposed enhancements aim to improve user compliance and decrease friction between citizens and traffic authorities.
